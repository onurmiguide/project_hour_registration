# Hours Tracker - Requirements & Installation Guide
# =============================================================================
# 
# SYSTEM REQUIREMENTS:
# - Windows 10/11 (or macOS/Linux)
# - Python 3.8 or higher (https://www.python.org/downloads/)
# - Visual Studio Code (already installed)
# - Internet connection (for CDN libraries and API calls)
# 
# INSTALLATION STEPS:
# =============================================================================
#
# STEP 1: Install Python (if not already installed)
# -----------
# 1. Download Python from https://www.python.org/downloads/
# 2. Run the installer
# 3. IMPORTANT: Check "Add Python to PATH" during installation
# 4. Click "Install Now"
# 5. Verify installation by opening PowerShell and running:
#    python --version
#
# STEP 2: Set up the project folder
# --------
# 1. Navigate to your project folder in PowerShell:
#    cd "c:\Users\OnurYilmaz\Desktop\stage miguide\hour registration\hour-tracker"
#
# STEP 3: Create a Python Virtual Environment (RECOMMENDED)
# -----------
# This keeps project dependencies isolated from your system
# 
# In PowerShell, run:
#    python -m venv venv
# 
# Activate the virtual environment:
#    .\venv\Scripts\Activate.ps1
# 
# If you get an execution policy error, run:
#    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
#
# STEP 4: Install Required Packages
# ---
# With virtual environment activated, run:
#    pip install -r requirements.txt
#
# STEP 5: Run the Application
# --
# Still in PowerShell with venv activated:
#    python app.py
#
# Then open your browser and go to:
#    http://localhost:5000
#
# STEP 6: Stop the Server
# --
# Press Ctrl+C in PowerShell
#
# =============================================================================
# TROUBLESHOOTING
# =============================================================================
#
# Problem: "python is not recognized"
# Solution: Restart PowerShell or add Python to PATH 
#           (may need to reinstall Python with "Add to PATH" checked)
#
# Problem: "pip is not recognized"
# Solution: Try using "python -m pip install -r requirements.txt" instead
#
# Problem: "permission denied" when activating venv
# Solution: Run PowerShell as Administrator or change execution policy
#
# Problem: Port 5000 already in use
# Solution: Change port in app.py or stop other processes using port 5000
#
# Problem: IndexedDB not working or files not saving
# Solution: Make sure you're using a modern browser (Chrome/Firefox)
#           Close all other tabs with the same localhost:5000 address
#

# =============================================================================
# FILE UPLOAD STORAGE
# =============================================================================
#
# Files are stored locally in the browser using:
#
# 1. localStorage (5-10MB)
#    - Stores file metadata (name, size, type, date, folder)
#    - Cannot store binary data directly
#    - Synchronous access
#
# 2. IndexedDB (50MB+)
#    - Stores actual file Blobs
#    - Asynchronous access
#    - Can store large files
#    - Per-domain quota
#
# Note: This is NOT a server-side storage solution
# Files are stored only in your browser's local storage
# Clearing browser data will delete all uploaded files
#
# To add server-side storage in the future:
# - Add backend file storage (e.g., Flask endpoint to save files)
# - Use SQLite or other database
# - May need to modify app.py
#
